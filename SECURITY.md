# 安全策略

## 报告漏洞

请优先使用 GitHub 仓库的 Private Vulnerability Reporting 或 Security Advisory 私下提交安全问题。不要在公开 Issue 中发布可直接利用的细节、恶意样例或敏感数据。

报告中请包含：

- 受影响版本或提交
- 复现条件和最小复现脚本
- 实际影响
- 建议修复方向（如有）

维护者确认问题后，会在修复可用时协调公开时间。

## 支持范围

安全修复以默认分支和最新公开版本为主。历史二进制归档不在支持范围内。

## 安全边界

本项目会执行用户选择的 BAT/CMD 文件。批处理拥有与启动用户相同的权限，启用管理员模式后权限更高。

请注意：

- 只转换和运行可信脚本。
- 不要在脚本中存放密码、令牌或私钥；生成 EXE 不会加密脚本内容。
- 不要把本工具当作沙箱、权限隔离或恶意代码检测工具。
- 不提供杀毒软件规避、隐蔽执行或持久化能力。
- 发布生成程序时建议使用代码签名，并通过可信渠道提供校验值。
- 安全软件可能对释放脚本、启动 `cmd.exe` 或管理子进程的行为产生误报，这不应通过规避安全产品来解决。
- 测试样例应保持可读、短时和可终止；禁止为了验证进程清理而构造隐藏 PowerShell 无限循环或编码载荷。

## 不属于漏洞的情况

- 从生成 EXE 中恢复原始批处理内容。
- 用户主动打包并运行恶意脚本。
- 未签名生成程序被安全软件拦截。
- 启用管理员模式后脚本获得管理员权限。

这些行为属于项目明确说明的工作方式或使用风险。
